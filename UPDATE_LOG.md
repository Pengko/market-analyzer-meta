# 📊 数据更新记录系统

用于追踪 Tushare 接口数据首次可用的时间，测试接口更新时间。

## 功能

- ✅ 只记录成功状态（失败/空数据不记录到文件）
- ✅ 记录首次成功获取数据的时间
- ✅ 每个接口只保留最新一条成功记录
- ✅ 自动生成 Markdown 格式记录文件

## 使用方式

### 1. 正常更新（自动记录）

```bash
# 正常更新会自动记录成功状态
python update_daily.py daily --date 20260410

# 更新后查看记录
python update_daily.py --log
```

### 2. 查看记录摘要

```bash
# 查看所有接口记录
python update_daily.py --log
```

输出示例：
```
📊 数据更新记录摘要
================================================================================

🔹 daily
  最新日期: 20260411
  首次成功: 16:35:22
  记录数: 5500

🔹 rt_idx_k
  最新日期: 20260410
  首次成功: 16:32:45
  记录数: 300

================================================================================
```

### 3. 轮询模式（测试接口何时可用）

当你想知道某个接口几点几分开始有数据：

```bash
# 每30秒尝试获取一次数据，直到成功
python update_daily.py daily --date 20260411 --poll
```

输出示例：
```
🔄 轮询模式: daily 20260411
每30秒尝试一次获取数据，记录首次成功时间

[16:15:00] ⚪ 第1次尝试: 无数据
[16:15:30] ⚪ 第2次尝试: 无数据
...
[16:35:22] ✅ 第42次尝试: 成功! 记录数: 5500
   🎉 首次获取到数据!
```

## 记录文件

记录存储在：**`~/quant-data/tushare/update_records.md`**

Markdown 格式示例：
```markdown
# 📊 数据更新记录

> 生成时间: 2026-04-12 17:45:36
> 记录规则: 每个接口只保留最新成功记录

---

## 🔹 daily

### 20260411

- **状态**: ✅ success
- **首次成功**: 2026-04-12T17:35:22
- **记录数**: 5500
- **检查次数**: 3

---

## 🔹 rt_idx_k

### 20260410

- **状态**: ✅ success
- **首次成功**: 2026-04-12T16:32:45
- **记录数**: 300
- **检查次数**: 5

---
```

## 记录规则

1. **只记录成功**：失败和空数据不写入文件（只在日志中显示）
2. **只保留最新**：每个接口只保留最新日期的成功记录
3. **自动更新**：新日期成功时自动替换旧记录

## 常用场景

### 场景1：查看最新数据更新时间

```bash
python update_daily.py --log
```

### 场景2：测试接口更新时间

```bash
# 收盘后启动监控
python start_monitor.py daily 20260411

# 查看记录文件
cat ~/quant-data/tushare/update_records.md
```

## 提示

- 记录文件是 Markdown 格式，可直接阅读
- 失败记录不会保存到文件，只在终端/日志中显示
- 新日期成功后会自动替换旧记录
