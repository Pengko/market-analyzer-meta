# 股票深度分析检查清单

在输出前，至少自检一次：

- 是否先确认当前中国时间
- 是否优先尝试获取中国网络时间，失败后才退回本地时间
- 是否根据时间正确识别为盘前 / 上午盘中 / 午间休盘 / 下午盘中 / 盘后
- 标的是否确认无误，名称和代码是否一致
- 是否说明数据日期或时间范围
- 是否在报告中标明时间来源
- 是否在报告中标明关键数据来自本地还是网络渠道
- 是否明确说明今日日内走势来自浏览器
- 是否明确说明今日最新日K来自浏览器
- 是否先用浏览器确认当天交易事实，再用本地数据确认是否已同步
- 是否区分 `浏览器已确认当天事实` 与 `本地数据已同步到当天`
- 是否先分析了市场，再分析板块，再分析个股
- 是否覆盖月线、周线、日线、分时
- 是否扫描了日K线中的所有缺口（向上/向下），并标注已回补/未回补
- 是否将缺口作为支撑/压力与均线并列使用（缺口优先级高于均线）
- 是否识别了缺口对峙格局（股价夹在两个未回补缺口之间）
- 是否拆解了 4 个关键时窗：09:30-09:35、11:25-11:30、13:00-13:30、14:55-15:00
- 是否检查了 `09:48-09:56` 首次强冲后的承接质量
- 如果是盘后分析，是否补看收盘集合竞价 `stk_auction_c`
- 如果是午间休盘或上午盘中，是否明确跳过尾盘集合竞价判断
- 如果用户问"能不能买"，是否给出了结构化入场方案（非模糊建议）
- 入场方案是否包含盈亏比计算，且盈亏比≥1.5:1
- 是否同时列出了多头和空头因素（不能只说一边）
- 是否明确了确认信号和失效信号
- 是否给出了具体仓位建议（不能只说"可以考虑"）
- 如果使用竞价数据，是否先确认样本落在合法竞价时间窗内
- 是否先判断该股是否为融券股，并分析融资融券强弱
- 是否覆盖资金、筹码、技术因子、消息面
- **资金流向是否为强制覆盖**：有本地moneyflow_data则用本地，无本地数据则必须通过东方财富API获取（`push2.eastmoney.com/api/qt/stock/fflow/kline/get`）
- 资金流向是否做了分时段增量分析（冲高/炸板/震荡/午前各段），而非只看全天累计
- 资金流向是否区分了超大单(>500万)和大单(100-500万)的行为差异
- 近5日每日资金流是否做了趋势对比（识别主力从出货转为回补等关键转折）
- **东方财富资金流字段映射是否正确**：f52=主力,f53=小单,f54=中单,f55=大单,f56=超大单，且验证 主力=f55+f56
- 无本地cyq_perf时是否用近10日VWAP估算了筹码集中区，并标注为估算
- 如果是基本面分析，是否包含风险背调（诉讼/监管/质押/关联交易/财务异常/信披/环保/舆情）
- 背调是否至少完成了5分钟快速版（审计意见+大股东质押+诉讼处罚+商誉减值）
- 若无 API，是否主动通过浏览器获取消息面
- 是否至少检查 `公告 / 政策 / 主流媒体 / 行情平台` 这 4 层渠道
- 是否在报告中写出已检查的消息来源渠道
- 是否明确给出 `消息方向 / 级别 / 新鲜度 / 可信度 / 情绪作用方式`
- 是否区分“新催化”与“旧消息重炒”
- 是否区分“公告实锤”与“传闻/二手转述”
- 是否判断消息面对走势是核心驱动、辅助驱动还是噪音
- 是否标注各类关键数据是 `可用 / 过期 / 缺失`
- 是否区分事实、推断、假设
- 是否给出次一交易日和后续几日预期
- 是否明确回答“当天是否适合买入”
- 是否提出反证或可能推翻观点的条件
- 是否给出后续跟踪指标

## 常用证据

- 时间上下文：中国网络时间或本地时间（需标明来源）
- 时段上下文：盘前、上午盘中、午间休盘、下午盘中、盘后
- 今日日内走势来源：浏览器网络渠道
- 今日最新日K来源：浏览器网络渠道
- 最新交易日判定：浏览器确认当天事实 + 本地数据同步确认
- 市场强弱：指数结构、涨跌家数、成交额、市场资金流
- 板块强弱：板块资金流、龙头强度、跟风分化
- 个股趋势：月/周/日位置、量价关系、收盘结构
- 分时主力：开盘抢筹、回落承接、尾盘回流、异常放量
- 关键时窗：09:30-09:35、09:48-09:56、11:25-11:30、13:00-13:30、14:55-15:00
- 收盘竞价：尾盘抢筹或兑现、竞价成交额、竞价均价与收盘价关系
- 融资融券：融资余额、融券余额、净变化、多空力量对比
- 资金流向：超大单/大单/中单/小单分时段增量、近5日趋势、主力从出货转回补等关键转折、分时段(冲高/炸板/震荡/午前)增量对比
- 筹码结构：筹码峰、筹码上移、主力成本区、获利盘/套牢盘、近10日VWAP套牢区估算
- 技术验证：趋势、动量、波动率、超买超卖
- 消息催化：公告、政策、行业催化、新闻热度
- 消息来源渠道：公告 / 官方 / 主流媒体 / 行情平台 / 社区传播
- 消息面情绪：偏利多 / 偏利空 / 中性 / 多空交织
- 消息可信度：公告实锤 / 主流媒体 / 二手转述 / 未证实传闻
- 消息作用方式：个股直驱 / 板块带动 / 市场情绪扩散 / 噪音
- 数据新鲜度：分钟线、竞价、筹码、资金、技术因子的最后日期与状态

## 常见误判

- 不确认当前中国时间，直接套错盘前/盘中/盘后逻辑
- 午间休盘却输出尾盘判断，或盘前假装已有盘中事实
- 把盘中或盘后快照误当开盘竞价或尾盘竞价
- 只看某一段流入流出，不看四个关键时窗前后是否一致
- 忽略 `09:48-09:56` 第一次试盘后的承接，把弱修复误判成强修复
- 使用本地缓存数据却误写成实时网络数据
- 只看本地日期，不先确认浏览器当天事实
- 浏览器已确认当天交易事实，却因本地日线滞后而误判成非最新交易日
- 还在依赖 `rt_min` 作为今日走势主入口，导致额度或时差问题
- 只看个股，不看市场和板块
- 把概念标签当主营业务
- 把短线情绪当成趋势反转
- 把盘中冲高误判为主升，忽略尾盘兑现
- 忽略融券压力，把表面强势误判为真实强势
- 只看融资余额，不看融资买入额和融券卖出量的增量变化
- 只看资金流入，不看筹码是否松动
- 只看指标金叉，不看所处位置和环境
- 只列利好，不列风险
- 只罗列消息标题，不判断消息情绪和可信度
- 没写消息来源渠道，导致后续无法判断证据等级
- 把旧消息重炒误判成新催化
- 把传闻当成实锤，把板块情绪当成个股独立逻辑
- 用户问"能不能买"时只给方向判断（偏多/偏空），不给结构化入场方案
- 遗漏缺口分析，只看均线不看缺口支撑/压力
- 遗漏资金流向分析（大单/超小单），只看成交量和涨跌幅 → **2026-06-12 实战教训：遗漏资金流导致将"超大单主导涨停"误判为"散户追高被套"，结论方向性错误**
- 东方财富资金流API字段映射搞错（f52=主力,f53=小单,f54=中单,f55=大单,f56=超大单），导致多空判断反转 → **必须用「主力=大单+超大单」验算确认映射正确**
- 无本地数据时跳过筹码分析 → 应用近10日VWAP估算筹码集中区并标注为估算
- 给出的入场方案没有计算盈亏比，无法判断是否值得做
- 只列利好不列风险，或只列风险不列利好

## 操作注意事项（实战经验）

### 数据字段解读

**筹码分布 (cyq_chips)**：
- `percent` 字段是**该价格点的筹码占比百分比**，不是累计百分比
- 计算获利盘/套牢盘时，需按价格与收盘价比较后分别累加
- 示例：`{'price': '4.1', 'percent': '21.4'}` 表示4.1元处筹码占比21.4%

**技术因子 (stk_factor_pro)**：
- 字段名使用 `ma_bfq_5`、`ma_bfq_10`、`ma_bfq_20`、`ma_bfq_30`、`rsi_bfq_6`
- 不是 `ma5`、`ma10`、`rsi_6` 等简化命名
- 其他字段：`pct_chg`（涨跌幅）、`volume_ratio`（量比）、`turnover_rate`（换手率）

### 沙盒环境限制

- **pandas 不可用**：Hermes 沙盒环境默认无 pandas，使用标准库 `csv` 模块替代
- 读取 CSV 时使用 `csv.DictReader`，按需转换数值类型
- 排序使用 `rows.sort(key=lambda x: x['field'])` 而非 DataFrame 排序

### 数据目录确认

执行分析前先确认关键数据目录是否存在：
- 概念成分表目录可能不存在（`dc_concept_cons`、`kpl_concept_cons`）
- 此时需通过浏览器或行情API获取板块信息作为降级方案
