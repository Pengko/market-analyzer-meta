# 公司治理变动分析框架

> 公司治理变动分析框架，覆盖三类事件：
> - **第一类**：重大人事变动（董事长/总经理辞职+新任命）
> - **第二类**：控股股东股权冻结/司法处置/质押违约
> - **第三类**：控股股东破产重整（控制权变更概率最高，不确定性持续最久）

## 触发条件

- 用户主动提供外部公告链接（如10jqka、东财、巨潮公告页）
- TrendRadar/消息面管线漏掉的同日新发公告
- 分析完成前用户发现遗漏的重要公告
- **控股股东股份被冻结/质押/司法处置类公告**（本框架第二类事件）
- **控股股东被申请破产重整/清算**（本框架第三类事件）

## 执行步骤

### 1. 读取公告全文

- 同花顺公告页（`news.10jqka.com.cn/tapp/notice.html`）为 Vue SPA，**必须通过浏览器工具**读取
- 东财/巨潮公告页可用 curl 或浏览器
- 提取关键字段：离任人、离任原因、新任人、新任人简历、控股股东推荐人

### 2. 新任人背调拆解

按以下维度拆解简历，不要只做流水账式复述：

| 维度 | 关注点 |
|------|--------|
| 技术出身 | 是否有行业技术背景（IC设计、封测、材料等），还是纯行政出身 |
| 管理轨迹 | 党务→业务→金融→投资 的路径分析，每段经历的战略意图 |
| 控股股东关系 | 是否由控股股东推荐/派遣，是否在控股股东体系内任职 |
| 产业链关联 | 是否同时在关联公司（如兄弟公司、上下游公司）担任董事 |
| 年龄与任期 | 年轻干部（<40）通常意味着控股股东长期战略布局 |

### 3. 战略信号解读

| 信号类型 | 判断方法 |
|----------|----------|
| 产业链整合信号 | 新任人同时在上下游关联公司任职（如设计+封测） |
| 控股股东加码信号 | 新任人是控股股东高管/核心干部 |
| 资产注入预期 | 新任人有投资/金融平台操盘经验 |
| 内部提拔信号 | 新任人已在公司任党委书记/总经理等职务 |
| 空降外来信号 | 新任人与控股股东/公司无历史关联 |

### 4. 修正概率分布

收到新公告后，需要对原分析的次日预期做修正：
- 评估新消息是**对冲卖压**（偏正面）还是**加重卖压**（偏负面）
- 修正原概率分布，标注 `原分析X%→修正后Y%`
- 若公告盘前发布且分析在公告前完成，需标注 `此公告发布于分析完成后，以下为补充修正`

### 5. 输出格式

在原报告基础上追加一个 `补充分析：公司治理变动` 章节（格式见模板区）。

## 已知陷阱

1. **公告时间 vs 分析时间**：盘前分析时当日公告可能尚未发布
2. **同花顺SPA页面**：curl/urllib 只能获取空壳HTML，必须用 `browser_navigate` + `browser_snapshot`
3. **简历流水账陷阱**：不要逐条复述简历，要提炼每段经历的**战略意图**
4. **工作调整≠利空**：国企"工作调整"可能是平调升迁也可能是边缘化
5. **关联公司任职≠利益输送**：控股股东体系内交叉任职是常态

## 第二类事件：控股股东股权冻结/司法处置分析框架

> 当控股股东股份被司法冻结、轮候冻结、质押违约、或进入司法拍卖时，按此框架执行。

### 分析步骤

1. **冻结/处置基本信息提取**：冻结主体、债权人身份、规模、类型、执行依据
2. **控股股东债务链条推演**：债权人数量、债务规模估算、其他资产、历史融资行为
3. **国资/战略投资者介入可能性评估**：公司战略价值、政府历史关系、债权人身份、市值与控股权、公司治理、政治时机
4. **对股价的影响路径**：短期恐慌抛压→中期不确定性压制→长期估值重估或持续承压
5. **关键跟踪信号**：轮候冻结数量、司法拍卖进展、战投引入动作、经营层声明、融资余额变化

### 已知陷阱

1. **"国资接盘"≠利好**：国资被动接管通常是追债回收
2. **轮候冻结的放大效应**：N轮冻结 = N个以上债权人排队
3. **反腐关联风险**：控股股东被拘留与官员调查关联时政治敏感度极高
4. **融资盘踩踏**：高融资余额个股易触发连环强平
5. **司法拍卖折价**：起拍价通常为市价7-8折，形成价格锚定压制

## 第三类事件：控股股东破产重整分析框架

> 当控股股东（或实际控制人）被债权人申请破产重整时，按此框架执行。破产重整意味着**控制权变更概率显著上升**，且涉及法院受理审查、债权人会议、重整计划表决等漫长流程，不确定性持续时间更长。

### 与第二类事件的关键区别

| 维度 | 第二类（冻结/拍卖） | 第三类（破产重整） |
|------|---------------------|-------------------|
| 法律阶段 | 保全/执行阶段 | 破产程序阶段 |
| 控制权变更概率 | 较低 | 较高 |
| 国资介入方式 | 被动接盘 | 作为重整投资人主动参与 |
| 影响持续时间 | 数周至数月 | 数月至1-2年 |
| 公司经营独立性 | 通常不受影响 | 可能进入预重整期限制 |

### 分析步骤

1. **破产重整基本信息提取**：申请人身份、债权金额、重整vs清算、受理状态
2. **控股股东股权结构拆解（核心必做）**：质押+冻结+破产三重叠加矩阵
3. **对上市公司的三层冲击评估**：控制权变更风险→质押股处置连锁→经营独立性
4. **历史案例对标分析（必做）**：至少找2-3个可比案例，对比持股/质押比例、白骑士介入时间、最大回撤、恢复周期
5. **多情景概率分布**：至少三情景+意外情景，总和=100%

### 已知陷阱

1. **"被申请"≠"已受理"**：法院可能不受理，未受理前影响有限
2. **小债权人博弈**：个人投资者申请大公司控股股东重整，往往是为了施压
3. **重整计划通过门槛**：需债权人过半数同意且代表2/3以上债权额
4. **白骑士引入不确定性**：从申请到计划可能6-18个月
5. **冻结+破产叠加**：处置需协调多个债权人和法院，时间极度拉长
6. **融资融券叠加**：高融资余额 = 强平踩踏风险
7. **"利空出尽"陷阱**：若个股已因其他利空暴跌，重整公告反而可能短期反弹，但不改中长期下行趋势
8. **重整 vs 清算的预期差**：从清算转重整 = 正面转折（有挽救价值）
9. **"破产重整"≠"公司破产"**：控股股东破产不等于上市公司破产

### 三个关键分叉点（决定股价走向）

**分叉1：市场预期是否已提前消化**

- **已提前消化**（如ST加加，公告前连涨5板+27.6%）→ 公告日即见顶，3日跌12.6%
- **未提前消化**（如方正系，公告前横盘）→ 法院受理日才是真正催化剂，受理日+5-6%

**分叉2：市场将事件解读为"利好"还是"利空"**

- **利好解读**（方正系）：控股股东巨亏→重整=引入新战投→法院受理日三股全跳涨
- **利空解读**（ST加加）：控股股东被受理破产清算→不确定增加→公告后连跌
- **利空出尽**（*ST亿晶）：公司已暴跌27%→重整=壳价值重估→公告日+4.4%

**分叉3：首次公告是"被申请"还是"法院受理"**

- **被申请阶段**（如方正系2019年12月）：影响有限，市场反应平淡
- **法院受理阶段**（如方正系2020-02-18）：影响最大，是真正的价格催化剂
- **核心逻辑**：法院受理 = 程序正式启动 = 资产处置进入快车道

### 按事件类型的预期模板

**模板A：控股股东被申请（未受理），市场尚未反应**
```
短期（1-3日）：影响有限，±2%波动
中期（1-4周）：取决于是否有更多信息泄露
关键催化：法院是否受理（参考方正系，受理日+5-6%）
最大风险：受理后若被市场解读为利空，则跌5-10%
```

**模板B：控股股东被申请，市场已提前反应（抢跑）**
```
短期（1-3日）：公告日即见顶，回落5-12%（参考ST加加）
中期（1-4周）：二次探底后可能反弹
关键催化：重整进展（清算转重整 = 正面转折）
最大风险：抢跑资金获利了结叠加恐慌抛售
```

**模板C：公司已因其他利空暴跌，破产重整公告出**
```
短期（1-3日）：利空出尽，可能+2-5%反弹（参考*ST亿晶）
中期（1-4周）：取决于是否有产业资本介入
关键催化：是否有白骑士/重整投资人
最大风险：壳价值重估后仍面临退市风险
```

## 快速查找历史案例公告日期的方法

### 方法1：东方财富公告搜索（推荐）

1. 访问 `https://so.eastmoney.com/ann/s?keyword={股票简称}+{关键词}`
2. 搜索关键词示例：`ST加加 破产清算`、`方正集团 破产重整`
3. 结果列表中可看到每条公告的日期和标题
4. 重点关注"法院裁定受理"类公告（这是最大的价格催化剂）

### 方法2：本地K线数据 + 大涨大跌日反推

当无法直接找到公告日期时：
1. 读取本地日线数据：`/Users/penghongming/quant-data/tushare/股票数据/daily/{year}/{code}.csv`
2. 找到异常涨跌日（涨跌>5%或成交量异常放大）
3. 对照新闻搜索确认该日是否有公告

### 方法3：Sina API获取近期K线（仅限近500个交易日）

```python
import urllib.request, json
url = f"https://quotes.sina.cn/cn/api/jsonp_v2.php/var/CN_MarketDataService.getKLineData?symbol={code}&scale=240&ma=no&datalen=500"
```
注意：Sina API最多回溯约500个交易日（约2年），更早的数据需用本地Tushare daily CSV。
