# 缺口分析框架 (Gap Analysis Framework)

> 2026-06-12 实战教训：分析红宝丽(002165)时遗漏了涨停缺口支撑，用户明确纠正。缺口是技术分析基本要素，必须与均线并列使用。

## 一、缺口定义

**跳空缺口**：相邻两根K线之间无价格重叠的区间，代表多空力量的突变。

- 向上缺口：`今日最低 > 昨日最高`，缺口区间 = [昨日最高, 今日最低]
- 向下缺口：`今日最高 < 昨日最低`，缺口区间 = [今日最高, 昨日最低]

## 二、缺口分类与意义

| 类型 | 特征 | 信号意义 | 回补概率 |
|------|------|----------|----------|
| 普通缺口 | 震荡区间内 | 无特殊意义 | 高(通常3-5日回补) |
| 突破缺口 | 趋势启动时 | 方向确认 | 低(趋势延续则不回补) |
| 持续缺口 | 趋势加速中 | 趋势强化 | 低 |
| 衰竭缺口 | 趋势末端 | 反转预警 | 高(很快回补) |
| **涨停缺口** | 涨停板后跳空 | **主力强意愿** | **低(重要支撑/压力)** |
| **跌停缺口** | 跌停板后跳空 | **恐慌抛压** | **高(持续压制)** |

## 三、分析流程

### Step 1: 识别所有缺口

```python
def find_gaps(klines):
    """klines: list of dicts with day/open/high/low/close"""
    gaps = []
    for i in range(1, len(klines)):
        prev_h = float(klines[i-1]['high'])
        prev_l = float(klines[i-1]['low'])
        curr_h = float(klines[i]['high'])
        curr_l = float(klines[i]['low'])
        
        if curr_l > prev_h:  # 向上缺口
            gaps.append({
                'type': 'up',
                'date_from': klines[i-1]['day'],
                'date_to': klines[i]['day'],
                'zone_lo': prev_h,  # 缺口下沿(支撑)
                'zone_hi': curr_l,  # 缺口上沿
            })
        elif curr_h < prev_l:  # 向下缺口
            gaps.append({
                'type': 'down',
                'date_from': klines[i-1]['day'],
                'date_to': klines[i]['day'],
                'zone_lo': curr_h,  # 缺口下沿(压力)
                'zone_hi': prev_l,  # 缺口上沿
            })
    return gaps
```

**注意**：新浪日K线API的scale=240才是日线，scale=5是5分钟线。

### Step 2: 判断缺口状态

对每个缺口，检查后续K线是否回补：
- **未回补**：后续所有K线的范围都未触及缺口区间
- **部分回补**：K线触及缺口但未完全覆盖
- **完全回补**：后续K线完全覆盖缺口区间

### Step 3: 按重要性排序

优先关注（权重从高到低）：
1. **近5日内的涨停/跌停缺口** — 主力资金刚介入或出逃
2. **近30日内的突破缺口** — 趋势启动信号
3. **近60日内的未回补缺口** — 中期支撑/压力
4. **远期已回补缺口** — 参考价值低，仅标记

### Step 4: 缺口对峙分析

当股价处于两个未回补缺口之间时：

```
上方缺口(压力): X.XX~X.XX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
现  价:        X.XX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
下方缺口(支撑): X.XX~X.XX
```

判断要点：
- 距离哪个缺口更近？
- 缺口之间是否有密集成交区？
- 缺口形成时的成交量大小（放量缺口更有效）

## 四、缺口与交易决策

### 支撑缺口的入场条件

| 条件 | 说明 | 权重 |
|------|------|------|
| 缺口未回补 | 多头防线仍有效 | 必要 |
| 缩量回踩 | 回踩缺口时不放量 = 筹码稳定 | 加分 |
| 分时托单 | 缺口附近有明显买盘承接 | 加分 |
| 缺口形成时放量 | 代表主力真金白银建仓 | 加分 |
| 涨停缺口 | 涨停板后形成的缺口 = 最强支撑 | 强加分 |

### 入场/止损设置

```
入场位: 缺口上沿附近 (保守则等放量收复缺口上沿后)
止损位: 缺口下沿下方 1-2% (跌破缺口 = 多头失败)
目标位: 上方最近压力位 (向下缺口下沿 / 前期平台 / MA)
```

**盈亏比计算**：
```
盈亏比 = (目标位 - 入场位) / (入场位 - 止损位)
要求盈亏比 >= 1.5:1 才值得入场
```

### 支撑失败信号（禁止入场）

- 放量跌破缺口下沿
- 连续2日收盘低于缺口下沿
- 缺口附近无反弹直接下杀

## 五、实战案例

### 案例：红宝丽(002165) 2026-06-11~12

```
6/10: 收7.30 (一字涨停)
6/11: 跳空高开7.72→收8.03 (再次涨停)
       → 向上缺口: 7.30~7.50 (涨停缺口，极强支撑)
6/12: 跳空低开7.44→现7.58
       → 向下缺口: 7.85~8.03 (压力)
```

分析结论：
- 股价夹在两个缺口之间(7.30~7.50 vs 7.85~8.03)
- 今日最低7.44已刺入支撑缺口区间，测试支撑
- 若守住7.30~7.50 → 可轻仓试探，止损7.25
- 若跌破7.30 → 涨停缺口回补，反弹失败，禁止入场

## 六、常见陷阱

1. **只看均线不看缺口**：缺口支撑/压力往往比均线更精确、更及时
2. **混淆5分钟K线与日K线**：新浪API scale=5是分钟线，scale=240才是日线，用分钟线找缺口会得到大量无意义缺口
3. **忽视缺口形成时的成交量**：无量缺口容易回补，放量缺口更可靠
4. **把已回补的缺口当支撑**：已回补的缺口失去支撑/压力意义
5. **忽略缺口对峙格局**：两个缺口之间的"夹心"结构是短线博弈的关键信息
