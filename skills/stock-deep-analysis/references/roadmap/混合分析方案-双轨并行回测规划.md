# 混合分析方案：双轨并行 + 差异对比 + 策略因子回测

> 状态：待细化  
> 创建时间：2026-04-22  
> 目标：建立可回测的规则引擎，同时保留AI自由分析能力，通过差异对比持续优化策略因子

---

## 一、方案概述（已定稿）

### 核心思想
同一组数据，同时走两条轨道生成分析结论，对比差异：

- **轨道A：AI自由撰写** → 深度自然语言报告（直觉/经验模式）
- **轨道B：规则引擎 → AI翻译** → 结构化信号报告（逻辑/量化模式）

**差异的价值**：
- 高度一致 → 高可信度，直接归档
- 方向一致但强度不同 → 校准规则敏感度
- **方向冲突** → **最有价值**，要么发现规则未编码的新模式，要么发现AI的叙事陷阱

### 为什么适合回测策略因子
用户核心目标是"跑大量历史数据，筛选准确率最高的策略因子"。
- 规则引擎提供**可量化、可复现**的信号
- AI自由分析提供**人类直觉基准**
- 两者差异数据本身就是优化规则的输入

## 二、架构设计（待细化）

```
历史数据 / 实时数据
        │
        ├── 数据层（quick_analyze.py 已就绪）
        │       输出：标准化 JSON（行情、分时、K线、因子、筹码）
        │
        ├── 轨道A：AI自由分析
        │       输入：原始数据 + 上下文
        │       输出：自然语言报告（含隐含观点）
        │       解析：extract_ai_signal() 提取隐含信号
        │
        ├── 轨道B：规则引擎
        │       输入：相同标准化 JSON
        │       处理：多维度加权打分 → 标准化信号
        │       输出：显式信号标签 + 置信度 + 人类可读翻译
        │
        └── 对比层
                ├── 信号对齐：轨道A隐含信号 vs 轨道B显式信号
                ├── 差异分类：一致 / 强度差异 / 方向冲突
                └── 归档：差异记录 → 供策略优化agent消费
```

## 四、关键设计决策（待讨论细化）

| 决策点 | 候选方案 | 倾向 |
|--------|---------|------|
| 规则引擎语言 | Python（与现有脚本一致）/ Rust（性能） | Python |
| 规则配置方式 | 代码硬编码 / YAML配置 / 数据库存储 | YAML + 代码混合 |
| 信号分级 | 3档（买/观望/卖）/ 5档（强烈买/买/观望/卖/强烈卖） | 5档 |
| 回测周期 | 1个月验证 / 3个月验证 / 1年验证 | 3个月验证，1年统计 |
| 差异处理方式 | 人工review / 自动记录供优化agent / 冲突时报警 | 自动记录 + 定期review |
| AI翻译触发 | 每次必翻译 / 仅差异时翻译 / 可选 | 轨道B必翻译 |

## 六、风险与注意事项

1. **规则过拟合风险**：规则在历史数据上表现好，未来可能失效。需留验证集。
2. **AI翻译同质化**：如果AI翻译只是模板的换一种说法，价值不大。要求AI基于因子明细做实质性解释。
3. **差异分类的主观性**："方向冲突"容易判断，"强度差异"的阈值需要调试。
4. **计算资源**：批量回测100只票×3个月≈6000条记录，规则计算量可控，但AI自由分析6000份报告成本过高。回测阶段轨道A可采样而非全量。
